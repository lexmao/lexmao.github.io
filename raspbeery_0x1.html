树莓派(raspberry pi) 操作备忘（1）

今天想玩一下raspberry pi，把2年前玩的电动车完善以下，但是发现两个问题：

问题描述
  1. root密码忘记了，同时wifi配置失效....ssh远程链接不上
  2. HDMI链接到投影设备无响应（无信号），就是说在本机上没办法操作，盲打^.^？
解决问题

  1. 首先必须解决视频输出的问题，否则没办法本机操作...其他都是白搭

    google 后解决方案：

    Step 1 – Grab The SD Card

    Power down the Pi and remove the SD card. Insert it into your PC.

    然后修改SD卡中config.txt文件中的内容： hdmi_safe=1 overscan_left=-30 overscan_right=-30 overscan_top=-30 overscan_bottom=-30 hdmi_group=2 hdmi_mode=4 hdmi_drive=2 config_hdmi_boost=4

    保存，然后把SD卡重新放入raspbeery中....然后可以可以看到投影设备的输出，问题解决。 主要是检测HDMI输出设备兼容方面的问题，所以在配置上改一下。

  2. 修改密码

    raspbeery是linux内核，所以按传统的本机方式（lilo,grub,single...）应该就可以操作了。 我刚好看到google的文章，就按下面方式解决：

    Step 1 – Grab The SD Card

    Power down the Pi and remove the SD card. Insert it into your PC.

    Step 2 – Edit cmdline.txt

    init=/bin/sh If the original content was :

    dwc_otg.lpm_enable=0 console=ttyAMA0,115200 kgdboc=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline rootwait

    it should now look like :

    dwc_otg.lpm_enable=0 console=ttyAMA0,115200 kgdboc=ttyAMA0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 elevator=deadline rootwait init=/bin/sh

    Make sure it is all one line! Save the text file and eject the SD card from the PC.

    Step 3 – Reset the Password

    passwd root

    ....

    Now type the following commands :

    sync exec /sbin/init

    shutdown -h now

    修改完密码后，把config.txt中的内容恢复回去，插入重新启动就好了。
